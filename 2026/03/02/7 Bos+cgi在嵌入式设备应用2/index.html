3 编译sensor-params.c在VMware

3.1 sensor.json

主要是对PARAMS，camera这两大模块的配置，yb暂时不管

一：
传感器类型是TYPE里面的数据并且与PARAMS对应。

PARAMS里面有5个元素。

每个元素有8个分别是：WAY，TCP，USART，FREQ，ADDR，ECODE，INITVAL，NUM。

WAY只能是0或者1，与TCP和USART有关。

TCP里面有IP和PORT可以自由填写。

USART内部CHN，BAUD，PARITY可以自由填写。

FREQ可以自由填写。

ADDR与ECODE有关，假如ECODE有x个元素，ADDR就是从1到x。

ECODE可以自由的增加删除改写。 每一个ECODE元素对应3个INITVAL元素。
约定-9999的元素值不被写入json。

不能自由增加删除改下，因为ECODE对应元素不确定，代码里面是1对3写死了。

NUM是ECODE内部元素数量，页面上可展示但不可改写。

二：

camera里面有两个元素。

每个元素有4个分别是：ID，ECODE，INITVAL，NUM。

ID自由填写，文本类型。

ECODE内部数量不确定，要求可以增加和减少。

NUM是ECODE内部元素数量，页面上可展示但不可改写。

每一个ECODE元素对应2个INITVAL元素

3.3.1 问题1-控制台addRow is not defined

F12控制台报错 addRow is not defined，这通常意味着脚本块未被正确解析执行。可能的原因包括：

  1. JavaScript 中的中文注释 在某些浏览器或编码（GBK）下可能导致解析中断，虽然控制台未显示错误，但整个脚本块可能被跳过。
  2. 脚本位置：函数定义在 <head> 中，通常没有问题，但为了排除干扰，可以移至 <body> 底部。
  3. 不可见字符：CGI 输出时可能引入了隐藏字符（如 UTF-8 BOM 头），导致浏览器解析异常。 简化 JavaScript（去除中文注释）

新引入js代码，但要注意

  • 编码一致：确保 CGI 输出的 HTML 声明的字符集（如 GBK）与 JavaScript 中实际使用的字符编码一致。如果代码中使用了中文注释，最好统一使用 UTF-8 编码。
  • 避免特殊字符：注释中的某些字符（如未转义的引号、换行）可能意外破坏 JavaScript 语法，但概率较低。
  • 调试建议：在 CGI 开发中，若遇到 JavaScript 不执行，可先移除所有非英文注释（或改为英文），以快速排除编码问题。待功能稳定后，再根据需要调整编码或添加注释。
